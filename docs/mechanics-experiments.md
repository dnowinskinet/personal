# GriftOS Mechanics Experiment Backlog

These are scoped hypotheses, not roadmap commitments.

## Experiment A: Active Timer Acceleration

Translation: clicking or pressing an active Hustle icon accelerates the current cycle.

Smallest prototype: one active Hustle button reduces remaining time.

Risk: players feel forced to spam.

## Experiment B: Stronger Milestone Rewards

Translation: unit thresholds make old Hustles matter.

Smallest prototype: tune 10/25-unit rewards for Troll Network and Podcast Network only.

Risk: hidden multiplier soup.

## Experiment C: Leverage Purchase Proof

Translation: the Leverage tab gets one run-specific modifier purchase.

Smallest prototype: one global output modifier with explicit cost and visible math.

Risk: a single card becomes the start of an unapproved giant tree.

## Experiment D: Temporary Burst Event

Translation: brief market mania or narrative window.

Smallest prototype: one visible timed opportunity that temporarily boosts output.

Risk: FOMO and interruption.

## Experiment E: Hustle Specialization

Translation: declare one Hustle as the thesis.

Smallest prototype: choose one Hustle for a run-long output boost.

Risk: balance complexity.

## Experiment F: Rug Pull Tuning

Translation: make the first reset viable and desirable.

Smallest prototype: compare threshold/output changes through the balance simulator before changing code.

Risk: Rug Pull becomes either mandatory too early or unreachable.

## Experiment G: Net Worth Upgrade Surface

Translation: persistent Net Worth becomes a source of explicit strategic choices.

Smallest prototype: one non-spendable display-only benefit remains current; do not add spendable Net Worth until explicitly approved.

Risk: introducing a second prestige currency by accident.

## Experiment H: Adaptive Audio Assets

Translation: replace generated debug tones with real layered music and SFX assets.

Smallest prototype: one short UI cue and one looped base music layer, loaded through the existing manifest.

Risk: audio polish distracts from unresolved economy pacing.

## Experiment I: Founder Core Active Input

Translation: Founder Core becomes a controlled active mechanic.

Smallest prototype: no implementation yet. First decide whether the core should ever be clickable.

Risk: it becomes a generic central clicker and undermines Hustle identity.
